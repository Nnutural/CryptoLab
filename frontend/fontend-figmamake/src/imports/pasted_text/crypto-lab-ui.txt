## 1. Overall Goal

Create a polished, interactive, production-oriented frontend for **CryptoLab**, not just static mockups.

The app should look like:

> GitHub-style engineering clarity + university laboratory professionalism + cryptography teaching visualization.

It must avoid generic AI-generated SaaS visuals. Do not use cyberpunk styling, excessive gradients, dark admin dashboard patterns, glassmorphism, cartoon icons, 3D objects, neon glow, marketing landing-page aesthetics, or decorative clutter.

The platform is an academic cryptography workbench that allows students and instructors to interact with 15 hand-written cryptographic algorithms through a clean Web UI.

Core algorithm scope:

* Symmetric encryption: AES, SM4, RC6
* Modes: ECB, CBC, CTR, GCM
* Hashes: SHA-1, SHA-224, SHA-256, SHA-384, SHA-512, SHA3-256, SHA3-512, RIPEMD-160
* HMAC
* PBKDF2
* Encoding: Base64, UTF-8
* Public key: RSA-1024, ECC-160, ECDSA
* Security demos: ECB image leakage, ECDSA k reuse, RSA low exponent, PBKDF2 iteration impact
* Capstone scenario: secure file transfer using RSA + AES-GCM + SHA-256 + ECDSA

The frontend must remain compatible with the backend API described in the attached documents. All UI forms, request payloads, response displays, error states, and route names must match the documented API.

## 7. API Contract Requirements

Use this uniform API response model:

```ts
interface APIResponse<T> {
  code: number
  message: string
  data: T | null
  trace_id?: string
  timestamp?: string
}
```

Axios behavior:

* base URL: `/api`
* dev proxy target: `http://localhost:8000`
* request interceptor injects `Authorization: Bearer <token>` from auth store
* response interceptor unwraps `data` when `code === 1000`
* typed error is thrown when `code !== 1000`
* `401`, `4001`, or `4002` clears token and redirects to `/login`
* never call axios directly inside views; use `src/api/*`

Business code display:

* 1000 success
* 2xxx client parameter error
* 3xxx cryptographic operation failure
* 4xxx auth / permission / rate limit
* 5xxx server error
* show error code badges inside OutputPanel

Endpoint coverage must include all documented endpoints:

Auth:

* `POST /api/v1/auth/register`
* `POST /api/v1/auth/login`
* `POST /api/v1/auth/logout`
* `GET /api/v1/auth/me`

Symmetric:

* `POST /api/v1/symmetric/keygen`
* `POST /api/v1/symmetric/{algo}/encrypt`
* `POST /api/v1/symmetric/{algo}/decrypt`

Hash / HMAC / PBKDF2:

* `POST /api/v1/hash/{algo}`
* `POST /api/v1/hash/hmac/{algo}`
* `POST /api/v1/hash/pbkdf2`

Encoding:

* `POST /api/v1/encoding/base64/{op}`

RSA:
